## SW 패키징

### 1. 작업 순서

- 기능식별
- **모듈화** : **모듈(기능 단위로** 분해하고 추상되어 **재사용 및 공유가 가능 단위**)을 통해 제품 SW **성능 향상**시키고 시스템 통합, 시험 디버깅 및 수정 용이하게 하는 SW 설계 기법
- 빌드 진행
- 사용자 환경 분석
- 패키징 적용 시험
- 패키징 변경 개선



### 2. 형상 관리 (SCM) ★

#### (1) 정의

- SW 개발 과정에서 **변경 사항을 관리**하는 기법

- **CVS, SVN, Git** 등 도구 활용
- 역할
  - 관리 유용
  - 동시 개발 
  - 빠른 도구

#### (2) 형상 관리 **절차**

- **형상 식별**
  - 형상 관리 대상에 이름과 관리 번호 부여
  - **계층(트리) 구조로** 구분
  - 수정 및 추적 용이하게 만드는 작업
- 버전 제어
  - 업그레이드나 유지 보수 과정에 생성된 **다른 버전의 형상 항목 관리**
  - 이를 위해 **특정 절차와 도구를 결합**시키는 작업
- **형상 통제 (변경 관리)**
  - 식별된 형상 항목에 대한 변경 요구 검토
  - **현재의 기준선(베이스 라인)이 잘 반영**될 수 있도록 조절
  - 베이스 라인 : 소프트웨어 변경 통제 시점
  - 컴포넌트 저장소 : 인증받은 컴포넌트를 등록하는 저장소로 사용자가 손쉽게 컴포넌트를 이용 가능
- **형상 감사**
  - 기준선의 무결성을 평가하기 위해 확인, 검증, 검열 과정을 통해 공식적으로 승인
- **형상 기록 (상태 보고)**
  - 형상의 식별, 통제, 감사 작업 결과 기록 관리하고 보고서 작성

#### (3) 형상 관리 기능

- 체크인 : 개발자가 **수정한** 소스를 형상관리 **저장소로 업로드**
- 체크아웃 
- **커밋** : 개발자가 소스를 형상관리 저장소에 **업로드 후** **최종적으로 업데이트가** 되었을 때 **형상관리**에 **반영**하도록 하는 기능

#### (4) **리포지토리(repository) ★**

- 형상 관리 사용하는 용어
- 관리 대상을 형상 관리 시스템으로 일괄 전송하여 압축, 암호화한 후에 
- 파일의 현재 버전과 변경 이력정보를 저장하는 **저장소**

## SW 버전 관리 도구



### 1. 정의

- 제품 SW 신규 개발, 변경, 개선과 관련된 수정사항 관리



### 2. 유형

#### (1) 공유 폴더 방식 

> **로컬** 컴퓨터의 **공유 폴더에 복사하고 저장** 방식
>
> 변경사항을 **DB**에 기록하여 관리 

- RCS : 파일 잠금 방식으로 소스파일 수정은 한 명으로 제한하여 버전 관리
- SCCS
- PVCS
- QVCS

#### (2) 클라이언트 / 서버 방식

> 버전 관리 자료가 **중앙시스템(서버)**에 저장
>
> 모든 버전 관리는 **서버**에서 수행 (서버 문제 생기면 복구되기 전까지 다른 개발자와의 협업 및 버전 관리 중단)

- **CVS** : 서버와 클라이언로 구성되어 다수의 인원이 동시에 접근 가능하여 버전 관리 가능 
- **SVN(Subversion)** 
  -  **CVS 개선**
  - 아파치 소프트웨어 재단 발표
  - 모든 개발 작업 **trunk 디렉터리에서** 수행
  - **commit 할 때 마다 revision 1씩 증가 (커밋 실패시 롤백 가능**)
  - 클라이언트는 대부분 운영체제에서 사용되지만 서버는 주로 유닉스
  - 디렉토리 파일 **자유롭게 이동 가능**
- CVSNT
- Clear Case
- CMVC
- Perforce

#### (3) 분산 저장소 방식

> 버전 관리 자료가 하나의 **원격 저장소(여러 사람 협업 위해 버전 공동 관리)**와 분산된 개발자 PC의 **로컬 저장소(개발자가 실제 개발 진행하는 장소)**에 **함께 저장**되어 관리하는 방식
>
> 로컬 저장소에서 버전 관리가 가능하므로 원격 저장소 문제가 생겨도 작업 가능

- **Git**
  - **branch를 이용하면** 기본 버전 관리 틀에 영향 주지 않으멶서 다양한 형태 테스팅 가능
  - **SVN 저장소에 대한 이관 가능**
  - branch, check-out, commit 등 **로컬** 환경에서 형상 관리 가능
  - push, fetch, pull 등 **원격** 환경 변경 전송 가능
- Bitkeeper : SVN과 비슷한 중앙 통제 방식으로 대규모 프로젝트
- GNU arch
- DCVS
- Bazzar

## SW 매뉴얼



### 1. 설치 매뉴얼

- 사용자가 제품을 **구매한 후 최초 설치 시** 참조하는 메뉴얼 목차, 개요, 이력정보, 설치 메뉴얼 주석, 설치도구



### 2. 사용자 매뉴얼 

#### (1) 정의

- 개발이 **완료**된 제품 소프트웨어를 고객에게 전달하기 우해 **패키징하고 설치와 사용**에 필요한 **제반 절차 및 환경**에 대한 전체 내용 포함

#### (2) 작성 순서

- 기능 식별
- 사용자 화면 분류
- 사용자 환경 파일 확인
- 초기화 절차 확인
- 이상 Case 확인
- 최종 메뉴얼 적용

#### (3) 작성 항목 (문서 이력 정보) ★

-  버전, 작성자, 작성일, 검토나, 일시, 검수인 등 일자별 기록



### 3. SW 배포판

- **배포판 정보**를 포함하여 개발된 **컴포넌트 혹은 패키지**가 제품화된 것

## 릴리즈 노트

### 1. 정의

#### (1) 릴리즈

- 개발이 완성된 소프트웨어 출시 및 배초

#### (2) 릴리즈 노트

- 정의
  - 개발자가 개발 과정에서 정리된 릴리즈 정보를 **최종 사용자**인 **고객과 공유**하기 위해 작성한 문서
  - **개발팀에서 직접** 작성
  - **현재** 시제로 작성
  
- 항목
  - 테스트를 거친 기능 및 서비스, 개선 사항
  - 테스트 결과와 정보 포함
  
- 고려사항 

  - 개발팀 작성하여 명확, 정확 및 완전한 정보 제공

  - 중대한 오류가 발생하여 긴급하게 수정하는 경우 릴리즈 **버전을 출시**하고 버그 번호를 포함한 **모든 수정된** 내용을 담아야 한다
  - SW 기능 업그레이드를 완료한 경우 릴리즈 **버전을 출시**하고 릴리즈 노트를 작성
  - **사용자**로부터 접수된 요구사항에 의해 수정된 경우 **자체 기능 향상과는 다른 별도의** 릴리즈 버전으로 릴리즈 노트 작성

## 디지털 저작권 관리



### 1. DRM (Digital Right Management)

#### (1) 정의 

- 제품 소프트웨어 패키징 도구 : 패키징 시 디지털콘텐츠의 **지식 재산권**을 보호하고 관리하는 기능 제공, 안전한 유통 배포 보조 도구
- 저작권 보호 패키징 도구 
  - DRM : 콘텐츠 **불법복제 방지**를 위한 **디지털 저작권 관리 기술**
  - DOI (Digital Object Identifier) : 디지털 저작물에 특정 번호를 부여하는 일종의 바코드 시스템
  - MPEG - 21 : 멀티미디어 표준 규격
  - CSM : **콘텐츠 생산 전 활동**의 과정을 관리하는 기술

#### (2) 구성 요소

- **패키저** : 콘텐츠를 **메타** 데이터와 함께 **배포 가능한 단위**로 묶는 기능
- 콘텐츠 제공자 : 콘텐츠를 제공
- 콘텐츠 분배자 : 쇼핑몰 등을 통해 **암호화된 콘텐츠**를 제공
- **DRM 컨트롤러** : 배포된 콘텐츠의 **이용 권한** 통제
- **클리어링 하우스** : **키 관리 및 라이선스** 발급 관리
- 보안 컨테이너 : **원본**을 안전하게 유통하기 위한 **전자적 보안** 장치
